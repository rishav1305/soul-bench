## About

I build AI systems that run reliably in production, every day, without hand-holding.

My primary project is soul-os -- a 9,700-line AI-native operating system in Python and React that orchestrates autonomous agents, heals its own failures through hysteresis-based remediation, scans for security vulnerabilities, coordinates distributed devices via mesh networking, and manages its own deployment pipeline. It is not a prototype. It is battle-tested infrastructure that has been running continuously in production.

The problems I care most about sit at the intersection of reliability and autonomy. How do you give AI agents real tools without letting them cause damage? How do you coordinate multiple models when any single one can fail? How do you build self-healing that stays stable instead of oscillating? I have shipped production-grade answers to each: boundary-enforced agent execution, Mixture-of-Agents orchestration with circuit breakers, and hysteresis-gated remediation loops.

Around soul-os I have built an ecosystem of 31 projects, 20 open source -- covering agent safety (soul-agents), distributed networking with hub election (soul-mesh), JWT auth with token-family replay detection (soul-auth), model benchmarking with a custom efficiency metric (soul-bench), and an AI-powered outreach platform (soul-outreach). Stack: Python 3.11+, FastAPI, React 19, SQLite, ChromaDB, Docker, Tauri.

I am looking for AI engineering roles at research labs and applied AI companies. If your team is working on hard problems in agent infrastructure, reliability, or multi-model systems, let's talk.
