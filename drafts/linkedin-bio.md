## About

I build AI systems that run in production unsupervised.

My primary project is soul-os, a 9,700-line AI-native operating system written in Python and React. It orchestrates autonomous agents, heals its own failures using hysteresis-based remediation, scans for security vulnerabilities, manages distributed devices through mesh networking with hub election, and handles its own deployment. It is not a demo. It runs every day.

The technical problems I care about most sit at the intersection of reliability and autonomy: how do you let AI agents use tools without letting them do damage? How do you coordinate multiple models when any one of them can fail? How do you build self-healing that does not oscillate? I have shipped working answers to each of these -- boundary-enforced agent execution, Mixture of Agents with circuit breakers, and hysteresis-gated remediation loops.

Around soul-os, I have built an ecosystem of 31 projects, 20 of them open source. They cover agent safety (soul-agents), distributed networking (soul-mesh), JWT auth with replay detection (soul-auth), model benchmarking with a custom CARS metric (soul-bench), and an open-core AI outreach platform (soul-outreach). The stack is Python 3.11+, FastAPI, React 19, SQLite, ChromaDB, Docker, and Tauri for desktop.

I am looking for AI engineering roles at research labs and applied AI companies, and I take consulting engagements at $150-300/hr for teams building agent infrastructure. If you are working on hard problems in AI systems engineering, I would like to hear about it.
